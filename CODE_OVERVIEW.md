# Codebase Overview

This document provides a detailed explanation of all the files and the underlying code architecture for the Vision-Controlled Tetris project.

## 1. `index.html`
This is the main entry point for the web application. It sets up the UI layout using HTML5 tags and links the necessary styles and scripts.
- **UI Structure:** It creates a left panel for the camera feed (`#cameraCanvas`) and a right panel for the Tetris game stage (`#gameCanvas`).
- **Dependencies:** Loads the `Outfit` font from Google Fonts, links to `styles.css`, and importantly, loads `app.js` as an ES6 module.
- **Hidden Elements:** It contains a hidden `<video>` element (`#cameraVideo`) which is used behind the scenes to capture webcam frames for MediaPipe to process.

## 2. `styles.css`
Contains the styling rules for the application.
- **Design System:** It defines CSS variables for colors, typography, and spacing to keep the design cohesive. 
- **Layout:** Uses CSS Flexbox and CSS Grid to align the camera feed and the Tetris game side-by-side. 
- **Aesthetics:** Implements modern glassmorphism effects, shadows, rounded corners, and smooth transitions to make the UI look premium.

## 3. `app.js`
This is the core frontend application logic containing both the gesture recognition inference and the Tetris game engine.
- **Initialization:** Loads the MediaPipe Hand Landmarker model via WebAssembly. It requests webcam access and starts a rendering loop.
- **Gesture Loading:** Attempts to fetch `gesture_profiles.json`. If it exists, it uses your custom-trained machine learning model for static gestures. If not, it falls back to hardcoded heuristic rules (checking finger tip vs joint heights).
- **Hand Detection & Classification:** 
  - Extracts exactly 21 3D coordinates from the hand.
  - Normalizes the coordinates relative to the wrist to make the gestures scale and rotation invariant.
  - **Static Gestures:** Uses Euclidean Distance matching comparing the current hand against the `centroid` profile of "Palm", "Index", and "TwoFinger".
  - **Dynamic Gestures:** Keeps a history buffer of wrist movements (`gestureHistory`). If the wrist moves rapidly on the X or Y axis over a few frames, it triggers "Left", "Right", or "Down".
- **Game Engine (Tetris):** Implements a standard Tetris gameplay loop (spawning pieces, handling collisions with a matrix board, dropping pieces, clearing lines, and maintaining score). The gestures act as events to control `hardDrop`, `softDrop`, `rotatePiece`, and piece movement.

## 4. `train_gestures.py`
A Python utility script used to train your own custom static gesture models using your webcam.
- It uses the Python MediaPipe API and OpenCV to read camera frames and extract hand landmarks.
- **Training Process:** You press "1", "2", or "3" to select a class (Palm, Index, TwoFinger). You hold the gesture in front of the camera, and it collects 160 frames of coordinate samples.
- **Machine Learning Logic:** Once samples are gathered, it calculates the **mean centroid** of all coordinate samples. It also calculates a dynamic threshold based on the 92nd percentile of distances from the centroid. 
- **Output:** It packages these centroids and thresholds into a `.json` configuration file.

## 5. `gesture_profiles.json`
The output artifact generated by `train_gestures.py`. 
- Contains the metadata (`featureLength=42` being the 21 normalized X, Y coords).
- Contains the `centroid` array for each class, which represents the mathematical "ideal" shape of that gesture based on your training data.
- Contains the `threshold`, indicating how far a new gesture can deviate from the centroid before being rejected.

## 6. `requirements-gestures.txt`
A small requirement file specifying the dependencies required to run the Python training script:
- `mediapipe`
- `opencv-python`
- `numpy`

## 7. `hand_landmarker.task`
The pre-trained MediaPipe machine learning model binary (a TFLite model packaged into a Task file) responsible for detecting the 21 hand landmarks from an image. It is downloaded automatically by the python script or loaded over CDN in the web app.
